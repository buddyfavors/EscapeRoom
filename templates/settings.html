Settings

Edit lock code pools as JSON. Keys: digit3 (3 digits), letter5 (5 letters), digit4 (4 digits). Any pool may be an empty array [] if you are not using that lock type — pick counts on the Play screen up to how many codes you list here.

RFID tags

One 10-digit tag ID per line (leading zeros count). Blank lines and lines starting with # are ignored. When a listed tag is scanned the engine randomly decides whether it's a good scan (reveals one character on a random unsolved lock) or a bad scan (Gamemaster punishment). Harder difficulties bias the roll toward bad. Each tag can be used once per run. Save tags removes duplicate lines (same ID listed twice) and reloads the editor so you only see unique IDs. On the Pi, the reader sometimes fires twice in a row — the listener drops identical scans within about a second.

Wheel of punishments

After three bad codes in a row (wrong lock combinations and/or bad RFID punishment rolls; only opening a lock resets the count), the wheel spins and one line below is chosen at random — no punishment repeats in the same game. Lines starting with minigame launch a minigame (use minigame for random, or minigame:<slug> for a specific one — reaction-rush, whack-mole, rps, simon, pattern). Any other line is shown to the players as a text punishment.
